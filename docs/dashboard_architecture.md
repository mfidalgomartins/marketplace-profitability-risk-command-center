# Executive Dashboard Architecture

## Deliverable
Official self-contained offline dashboard:
- `outputs/dashboard/marketplace_command_center_dashboard.html`

Optional non-official sampled build:
- `outputs/dashboard/marketplace_command_center_dashboard_demo.html`

Builder script:
- `src/dashboard/build_executive_dashboard.py`

## Architecture Overview
The dashboard is generated by Python and rendered client-side in a single HTML file with:
- embedded Plotly runtime (inline JS)
- embedded JSON data payload
- embedded CSS/JS (no external assets)
- tab-based control-tower layout
- sticky executive header + filter bar
- filter-driven re-aggregation in-browser for exploratory slices
- governed executive KPI anchoring from `reports/executive_kpi_snapshot.csv`

This design supports offline portability while preserving high interactivity.

## Data Flow
1. Python loads raw and processed marketplace tables.
2. A denormalized order fact layer is assembled for filtering and chart computation.
3. Scenario and seller score snapshots are embedded.
4. Official executive KPIs are injected from governed snapshot artifacts.
5. JSON payload is injected into the HTML template.
6. JavaScript applies filters and recomputes exploratory slice metrics/charts.

## Performance Choices
- Single denormalized order-fact payload minimizes repeated joins in JS.
- Compact field names in embedded JSON reduce HTML size and parse overhead.
- Optional stratified sampling mode (`--max-orders`) reduces payload size for lightweight demos.
- One-pass aggregation (`computeState`) computes exploratory section metrics for current filters.
- `Plotly.react` updates existing chart containers for efficient redraws.
- Seller table uses search + sort + pagination (limited row rendering).
- Filters are explicit and synchronous to avoid expensive uncontrolled rerenders.

## Update Instructions
1. Regenerate source and processed layers if upstream data changed.
2. Rebuild scenario outputs.
3. Rebuild visualization outputs (optional for separate chart exports).
4. Rebuild dashboard:
```bash
.venv/bin/python src/dashboard/build_executive_dashboard.py \
  --raw-dir data/raw \
  --processed-dir data/processed \
  --reports-dir reports \
  --output-file outputs/dashboard/marketplace_command_center_dashboard.html
```
5. (Optional) Rebuild non-official demo dashboard:
```bash
.venv/bin/python src/dashboard/build_executive_dashboard.py \
  --raw-dir data/raw \
  --processed-dir data/processed \
  --reports-dir reports \
  --output-file outputs/dashboard/marketplace_command_center_dashboard_demo.html \
  --max-orders 25000 \
  --sample-seed 42
```
6. Open the HTML file locally in any modern browser.

## Validation Notes


- correct coverage dates and row counts
- non-empty KPI cards for default filter scope
